01 / OVERVIEW

專案簡介

本專案實踐 「端點感知 (Edge) + 邊緣伺服 (PC) + 雲端智腦 (NIM)」 的協同運算架構。STM32 負責即時 I2S 音訊擷取與硬體控制；繁重的 AI 推論交由 PC 中繼站與 NVIDIA 雲端服務完成。降低嵌入式裝置負擔之餘，亦實現極具人性化的智慧對話體驗。

Main Controller

STM32F407VET6

Cortex-M4F · 168 MHz
  • 時序管理：I2S2 Master Clock，全雙工錄放音同步。
  • DMA 雙緩衝：Ping-Pong Buffer + Ring Buffer，音訊不中斷。
  • 資料流控制：USART1 921600 封包，與 ESP32 持久連線。
  • UI 更新：透過 SPI 高速更新 OLED 對話狀態。
Comms Gateway

ESP32-WROOM-32E

Wi-Fi · 8 MB Flash
  • 網路維護：Wi-Fi 連線管理與容錯重連。
  • 透明傳輸：USART1 ↔ TCP Socket 持久連線橋接。
  • 流量整形：配合 PC Sliding Window 控制下行緩衝。
Edge Server

PC Local Server

Python · RTX 5060
  • ASR 辨識：faster-whisper large-v3 即時語音轉文字。
  • 雲端串接：NVIDIA NIM google/gemma-4-31b-it 推論。
  • TTS 合成：本地 GPT-SoVITS V2 (Koharu) 日/中文合成。
Audio I/O

Mic & Amplifier

ICS43434 · MAX98357A
  • ICS43434：I2S MEMS 數位麥克風（模組亦標示 INMP441，腳位相容），L/R=GND 取左聲道。
  • MAX98357A：Class D 功率放大，直接驅動 4Ω 3W 喇叭。
03 / WORKFLOW

製作流程

由低層硬體驗證逐步向上整合至 PC AI Pipeline，每一階段都以可重現的測試為驗收標準。

STM32 基礎驗證DONE

完成 GPIO、按鍵、USART1 921600 8N1 與 USB CDC log，確保板子、線路與基本除錯通道可靠。

I2S 音訊輸出與輸入DONE

先驗證 MAX98357A 播放，再驗證 ICS43434 收音。STM32 為 I2S Master，TX 持續餵資料維持 BCLK/WS。

Audio Capture / Playback LoopbackDONE

Stage 7 完成 buffered capture/playback 驗證，firmware 使用 I2S2 full-duplex DMA circular buffer。

ESP32 雙向音訊橋接DONE

Stage 8 完成 PCM1 錄音上傳與 AUD1 音訊回放穩定化：持久 TCP 連線、24 KB Sliding Window 流量控制、64 KB Ring Buffer 與淡出衰減消爆音。

PC AI Pipeline 實機驗收ACTIVE

assistant_server.py 與 translator_server.py 已串接 faster-whisper → NVIDIA NIM (google/gemma-4-31b-it) → GPT-SoVITS V2 (Koharu)，待硬體實測延遲基準與斷句驗證。

OLED UI 與穩定度收尾TODO

補上 OLED 對話狀態顯示、整體延遲調教，並完成連續對話穩定度測試與最終報告。

06 / STRATEGY

預期成果與測試策略

為降低開發風險，採「翻譯機模式優先」的降級測試策略，再逐步推進至「全功能對話模式」。兩種模式共用同一條音訊 / TTS 鏈路，僅切換 NIM 端的 Prompt 與 LLM 上下文。

PHASE 1 · 翻譯機模式

中低延遲「隨身翻譯機」

Prompt 簡單明確、無須維持對話上下文，適合初期驗證通訊與音訊轉換的穩定性。對應 translator_server.py。

  • 功能描述：按 K1 說中文，系統翻譯成日文並用 GPT-SoVITS V2 (Koharu) 自然敬語播報。
  • AI Prompt：你是專業日文翻譯員，將輸入中文直接翻成具備敬語的自然日文，不輸出解釋。
  • TTS：GPT-SoVITS V2 本地推論（RTX 5060 fp16，一句約 5 秒）。
PHASE 2 · 智慧助理模式

低延遲「智慧語音助理」（終極目標）

翻譯機模式於 STM32 上的通訊與播放穩定後，切換 Prompt 進入全對話模式。對應 assistant_server.py。

  • 流暢對話：按鍵與 NIM (google/gemma-4-31b-it) 進行多輪語音對話，目標回應低於 2.5 秒。
  • 雙語能力：支援中、日、英多國語言翻譯與語境分析。
  • 硬體視覺化：OLED 即時顯示音量、連線狀態與對話文字摘要。
